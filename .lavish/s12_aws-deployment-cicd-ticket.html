4 · Two optional cost/ops levers

Both raised as questions — captured here as options, not defaults.

🗄️ Aurora Serverless v2 instead of RDS?

Aurora Serverless v2 now scales to zero / auto-pauses — ~$0 compute when idle, and data persists through the pause. pgvector supported.

  • Cheaper only if idle most of the time vs flat ~$15/mo RDS t4g.micro.
  • ~10–15s resume latency stacks on the App Runner cold start → slow first hit after quiet spells.
  • Not run locally — local stays on the Postgres container; both rebuild from the same Alembic migrations, so it's fully reproducible / destroyable (terraform destroy).

Default: RDS t4g.micro (predictable). Switchable to Serverless v2 if "pay nothing when idle" > first-hit latency.

📊 Tracing without Logfire Cloud

You already trace into the DB: every /ask writes app.query_runs (trace + SQL + tokens/latency) and app.events — that's what the admin dashboard reads.

  • Logfire is optional and OpenTelemetry-based → export can point at CloudWatch / X-Ray (serverless) or dump spans to S3 (cheap, query with Athena).
  • Default for v1: keep the DB audit trail (already built, $0) + CloudWatch for container logs; skip Logfire Cloud.

5 · The build plan

🔗
Prerequisite: ticket s11 (Google auth) is done and verified locally. The auth code deploys here unchanged — this ticket only adds the cloud client ID + redirect URI.
  • A · AWS foundations (Terraform)

    TF state (S3 + DynamoDB lock), ECR repos, RDS Postgres 16 (enable pgvector), Secrets Manager entries, the GitHub-OIDC IAM role, and the S3 data bucket. No app yet.

    ✔ Verify: terraform apply clean; connect to RDS and CREATE EXTENSION vector;.

  • C · Build & ship images + static frontend

    Production frontend build → S3 + CloudFront (replaces the dev-server). Confirm the 4 Dockerfiles build for linux/amd64; push backend-api / data-agent / db-migrate / pipeline to ECR. Point the pipeline at S3 for the full CSVs instead of the bind mount.

    ✔ Verify: images in ECR; static frontend loads from CloudFront.

  • D · Deploy services + run jobs

    Terraform the two App Runner services (backend-api public, data-agent downstream), secrets from Secrets Manager, DB_SSL=require, CORS + Google redirect URI set to the CloudFront URL. Run the migrate then pipeline ECS tasks — loading the full datasets from S3.

    ✔ Verify: API health + /auth/config in the cloud; full marts populated in RDS; Google login works on the live URL.

  • E · CI/CD — one push-button workflow

    New deploy-aws.yml: OIDC into the IAM role → build/push all images → terraform apply → run the migrate task → trigger App Runner new deployment → run the smoke test against the live URL. Keep ci.yml as the PR gate.

    ✔ Verify: a merge to main deploys with no manual steps; cloud smoke green.

  • F · Domain, TLS & harden later

    Route 53 + ACM cert → custom domain (HTTPS) on CloudFront. WAF, statement timeouts, per-user LLM cost caps, CloudWatch alarms. Optional: graduate App Runner → ECS Fargate for a private agent. Finalise Google redirect URIs for the real domain.
