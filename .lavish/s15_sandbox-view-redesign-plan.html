② Sandbox — a "data pipeline" view for the Golden tab

The sandbox is the step that decides what's in the report: it takes the SQL extract, enriches it (average price, growth, moving averages…), and emits the report JSON that feeds every presentation object. Today that middle stage is invisible and hard to steer. This proposes making the whole flow legible extract → enrichment → objects — grounded in your real "suburbs by area band" run. Matched to the Golden tab's own design system.

01

What you can & can't see today

  • One "augmented output" table — only main_chart.data

✕ Invisible today — the important part

  • The derived tables the code built (band_agg, suburb_total…) — where avg_price is calculated
  • Which enrichment feeds which report object
  • Every object it produced, side-by-side with the data behind it

root cause: sandbox exec(code) then reads only result — every derived frame in the namespace is dropped (runner.py:144).

03

How it becomes editable — steering the report

A · edit the script
run_analysis stays the source of truth. The pipeline re-renders on ▶ Run, so a code change instantly shows its new derived frames and new objects.
B · promote a frame → object
Each derived frame gets a "＋ add as object" action (pick chart type + columns), so you build a presentation object from enriched data without hand-writing spec code.
C · per-object provenance
Every object shows its source frame + fields, so "change what's in the report" starts from a visible mapping rather than reverse-engineering JSON.

05

Risks & things to decide

Frame noise
Scratch/temp frames (band_detail, pivots) can clutter Stage 2. Decide: show all frames, or only ones referenced by an object?
Payload size
Big enriched frames → truncate to head(20) + a "N rows" badge (same as the extract table today). No full-frame transport.
Serialization edges
Datetime / NaN / MultiIndex columns need safe coercion (reuse the extract's jsonable path).
Scope of "editable"
A/B/C above are increasing effort. Start with A+view only, or include B ("add frame as object") in v1?
