Data Pilot — full frontend restyle

The app becomes Data Pilot: your data, flown right. This plan covers a new sign-in experience that introduces the product with a data-+-flight walkthrough, a complete theme redesign (3 candidates below — pick one), a fix list from testing every screen in both modes, and data-consumption upgrades across the app. Nothing is executed yet — review, annotate, choose.

2 broken surfaces (theme-blind) 14 issues logged 3 theme candidates 6 execution phases

Decisions so far

Walkthrough ✓ auto-advancing flight path (4s, pausable) Scope ✓ all phases A → F, in order Icons ✓ Bubble · Terminal · Reticle · Gauge · Sliders Logo ✓ flat airplane icon (45°) Titles ✓ agent 3–5 word summary (E1) Theme ✓ A · Night Flight (gold-on-navy)

All decisions locked. Building now, starting with phase A (rename → Data Pilot, Night Flight token sets, flat-airplane BrandMark, theme-color/favicon) in the codebase on the eval-loop branch.

01

Where the frontend stands today

Solid foundation

A real two-layer token system already exists in styles.css (primitives → semantic tokens, switched by <html data-theme>). Charts read tokens via ui/charts/tokens.ts. The restyle is a token swap + component adoption, not a rebuild.

Two surfaces never joined the system

Golden Examples is ~1,300 lines of inline styles (rgba(128,128,128,…) grays, native white inputs) — it renders as a white page inside dark mode. SQL editor hardcodes the oneDark CodeMirror theme — a dark island in light mode.

Brand is a placeholder

"Datapilot" (one word), a bare 400px card at sign-in, no product story, emoji icons in suggestion cards, a brand mark hardcoded to a dark tile. The name becomes Data Pilot and the login becomes the product's front door.

03

Pick the Data Pilot theme

click a card to preview — every POST mockup on this page re-skins

A · Night Flight

evolution
Cockpit-at-night. Keeps the gold identity but moves the canvas from neutral gray to a deep navy-ink, brightens the gold for instrument-panel warmth, light mode becomes warm chart-paper. The safest brand continuity.
Wins
  • Zero re-learning — current users see "the same app, sharpened"
  • Gold reads premium; warm light mode is distinctive vs generic SaaS white
Costs
  • Least "new product" impact; gold-on-navy needs care for AA on small text

B · Jetstream

recommended
Sky and contrails. Stratosphere-navy dark mode with an electric cyan accent; cloud-white light mode with an aviation blue. The strongest literal fit for "data + flying", and blue is the native color of trust + data tools.
Wins
  • Best brand-name fit; both modes feel equally first-class
  • Cyan/blue chart ramps are the most legible categorical family
Costs
  • Biggest visual break from today's gold; blue is a crowded SaaS space

C · Flightdeck

distinctive
Avionics + radar. Phosphor-green HUD accent on graphite, mono-spaced micro-labels, amber for warnings; light mode is sectional-chart cream. The most memorable and most "instrument panel" of the three.
Wins
  • Instantly recognisable; terminal-green flatters SQL/code surfaces
  • Radar motifs give the login walkthrough a strong visual device
Costs
  • Green accents can read "success" in data UIs; niche taste for business users

Login splits into two panels. Left: the sign-in card sits in a cockpit scene — canopy pillars and a HUD (heading tape + reticle) frame the view, a curved instrument dash with gauges sits at the base, and a jet traces a contrail past drifting chunks of data and algebra (Σx², SELECT *, y = mx + b…) over the faint nav-grid.
