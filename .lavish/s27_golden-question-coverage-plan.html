4 · How they get populated

Pack-first seeding, then the normal curation loop

The pack is already the designed seeding path (eval_pack.py: "seeds any env, reviewable in a PR"). Drafts are legal pack citizens: CI's pack lint only requires case_key / question / dataset / tier ∈ T1–T7, and skips the SQL & grader checks when those fields are absent. Verified against tests/test_eval_pack.py.

1
Author 30 minimal pack entries — append 10 to evals/cases/nsw_sales.yaml and nsw_rent.yaml, create nsw_yield.yaml. Each entry is question-only: no SQL, no report, no grader yet. 
- case_key: nsw_yield-trust-the-2840-yield-<uuid>
  question: Postcode 2840 shows a ~14% gross yield — can I trust that number?
  dataset: nsw_yield
  tier: T5
  as_user: user1
  holdout: false
  origin_env: dev
  authoring_status: draft
  expectation: >-
    Flags thin data — n_sold/n_rented near the ≥5 floor, remote-area
    volatility — and answers with the caveat, not just the number.
  tags: [coverage-pack, direction:adversarial]
2
Guard the runner — eval_run.py::load_cases currently loads every pack case; a draft with no golden_sql would still be replayed against the live agent and graded against empty ground truth (verified at eval_run.py:245). Small patch: skip authoring_status: draft unless --include-drafts. Without this, make eval burns 30 live agent runs per invocation.
3
Import → export round-trip — make eval-import upserts the 30 into app.eval_cases; make eval-export rewrites the YAML in canonical form (field order, capping) so the committed pack matches what the tooling produces. Then verify the Golden tab shows 30 DRAFT lamps across the three dataset filters.
4
Prefill all 30 with the agent (decision Q2) — for every draft, call Draft with agent (POST /admin/eval-goldens/draft) to generate a first-pass SQL / sandbox / report, then save it onto the golden. The rows stay authoring_status: draft — the prefill is a first pass to edit, not a promotion. Each carries an expectation line so future-you knows what "right" looks like; flipping draft → ready (grader spec + review) stays your loop.
5
Ship — commit pack + runner guard on golden-question-drafts, validate through /no-mistakes, PR.
